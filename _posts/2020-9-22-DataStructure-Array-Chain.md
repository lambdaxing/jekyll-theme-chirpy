&emsp;&emsp;数组和链表可以说是最为基础的数据结构，分别代表了两种数据结构的存储方式（顺序存储 和 链式存储）。

&emsp;&emsp;一个数组的每一个实例都是形如（索引，值）的数对集合，其中任意两个数对的索引都不相同。数组是大部分语言的标准数据结构，其模型类似下图：

![array_img](/assets/img/2020-9-22-DataStructure-Array-Chain/array_img.jpg)

有关数组的操作:

+ 取值： 对一个给定的索引，取对应位置上的值。
+ 存值： 将新值存入给定索引标识的位置。

&emsp;&emsp;数组存储于内存上一片连续的空间中，通过索引我们可以很容易在 O(1) 内实现随机访问。但是数组的内存空间是一次性分配的，扩容操作需要分配更大的空间，再将原数据拷贝到新空间中，时间复杂度是 O(n) 。由于连续紧凑的存储方式，插入和删除操作涉及拷贝以及移动元素，时间复杂度也是 O(n) 。总之，数组的顺序存储方式，为随机访问带来了便利，节省了存储空间，但也提高了进一步抽象的复杂性。  
&emsp;&emsp;链表是一种链式存储的表结构，该数据对象实例的每一个元素都用一个单元或节点来描述。每一个节点都明确包含另外相关节点的位置信息，这个信息称为链 ( link ) 或指针 ( pointer ) 。 元素在内存中的存储位置是随机的，元素与元素间都有明确的指针或链（即地址）来连接。链表的模型，大概类似于下图：

&emsp;&emsp;相比于数组，链表不存在扩容的问题，插入和删除的时间复杂度为 O(1) ，但是我们不能在 O(1) 内通过索引实现随机访问，存储指针也使得耗费更多的空间。除此之外，我们还可以自由定义链表的内容和操作，这给予了我们极大的灵活性，链表的组织形式也是多种多样：单向 or 双向，带 or 不带头节点，循环 or 不循环。这给了我们多种选择方案来构建更加高效、茁壮的数据结构。  
&emsp;&emsp;为什么说数组和链表是最为基础的数据结构呢？数据结构是数据对象的描述和相关操作的集合，最终都是要存储在存储器上的，存储方式也就是顺序存储和链式存储，而数组和链表不就是最基本的顺序存储和链式存储吗。在数组和链表上进一步抽象，我们可以构建出丰富多彩的数据结构世界。栈和队列可用数组描述，也可链式描述；散列表可用数组采用线性探查法，也可采用链式散列。树结构既可以由数组描述，也可以链式描述，但由于树结构丰富多变，采用链式描述更好。堆是一棵完全二叉树，操作比较简单，可由数组实现的完全二叉树描述；图的表示方法：邻接矩阵、邻接链表、邻接数组，矩阵就是二维数组，数组描述很容易判断连通性，对于稀疏图很耗费空间，链表描述在效率上低于数组描述。一个数组，在实际问题中，赋予其完备强大的抽象往往能解决很多复杂深奥的问题。

## Note

&emsp;&emsp;C++中的顺序存储和链式存储:

+ `array`、`string`、`vector` 将元素保存在连续的内存空间中，支持快速随机访问。但 `array` 大小固定，`string`在尾部插入/删除速度快，`vector` 在尾部之外的位置插入或删除元素可能很慢。
+ `list`、`forward_list` 不支持随机访问，在任何位置添加删除元素都很快速。`list` 支持双向顺序访问，`forward_list` 支持单向顺序访问。
+ `deque` 支持快速随机访问，在两端添加或删除元素很快。
